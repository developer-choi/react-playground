에러처리를 크게 몇가지로 나눠봤음.

1. 서버사이드에서 페이지 렌더링 하다가 에러나는 경우 (ex: 로그인안하고 어떤 페이지 간 경우)
   ==>
   Case1. 다른 페이지로 리다이랙트 (로그인 페이지로 리다이랙트)
   Case2. Alert로 경고창을 보여주고 리다이랙트 (없는 블로그로 이동한 경우 "삭제된 게시글입니다" 보여주고 블로그 메인페이지로 리다이랙트)
   Case3. 처리되지않은 에러인 경우에는 500에러 페이지 반환

2. 클라이언트사이드에서 에러나는 경우 (ex: 사이트 내 재화를 충전하려고 충전버튼을 눌렀을 때 로그인안하고 누른경우)
   ==>
   경우에 맞는 알림팝업으로 에러 원인을 알려주는 등의 피드백 진행.
